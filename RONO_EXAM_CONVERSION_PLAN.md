# نقشه‌راه تبدیل رونو به سیستم آزمون استخدامی

> سند برنامه‌ریزی برای تبدیل رونو (پلتفرم یادگیری تطبیقی پزشکی، فورک‌شده از Synapse)
> به **سیستم یادگیری و مرور سؤالات آزمون‌های استخدامی پرمخاطب ایران**.
> مبتنی بر گزارش تحقیق: [RONO_EMPLOYMENT_EXAM_REPORT.html](RONO_EMPLOYMENT_EXAM_REPORT.html)
>
> آزمون‌های هدف: دستگاه‌های اجرایی (فراگیر)، آموزش و پرورش، بانک‌ها، تأمین اجتماعی.

---

## یافته‌ی کلیدی معماری: ۸۰٪ زیرساخت از قبل آماده است

بک‌اند رونو از قبل تقریباً همه‌ی چیزی را که لازم داریم دارد. این کار عمدتاً
**«پیکربندی مجدد + بومی‌سازی + محتوا»** است، نه ساخت از صفر.

| نیاز ما | چیزی که از قبل هست | فایل |
|---|---|---|
| نوع آزمون | `item.exam_type` (ستون آزاد) | `models/item.py` |
| بخش آزمون (عمومی/تخصصی) | `item.exam_part` (الان: basic/clinical) | `models/item.py` |
| منشأ سؤال (کدام آزمون/سال) | `source`, `source_reference`, `exam_year`, `exam_session` | `models/item.py` |
| زبان محتوا | `item.language` (الان پیش‌فرض `tr`) | `models/item.py` |
| درس → مبحث (سلسله‌مراتبی) | `Topic` (parent_id, path, level, slug) | `models/topic.py` |
| تعریف هر آزمون | `Curriculum.target_exam` | `models/curriculum.py` |
| گزینه‌ها | `Option` (تعداد گزینه آزاد است؛ A–E الان، A–D هم می‌شود) | `models/option.py` |
| نمره منفی | مهاجرت `..._penalty` از قبل هست | `alembic/versions/f1a2b3c4d5e6_*` |
| **ورود انبوه سؤالات سال‌های قبل** | `scripts/import_questions.py` (JSON → سؤال) | `scripts/import_questions.py` |
| الگوی seed تاکسونومی | `scripts/seed_tus_topics.py` | `scripts/` |
| موتور مرور هوشمند | FSRS (`services/fsrs/engine.py`) | بدون تغییر، فقط پشت‌صحنه |
| موتور تطبیقی | IRT (`services/irt/engine.py`) | بدون تغییر، فقط پشت‌صحنه |
| نسخه‌بندی، تگ، فلگ سؤال، کارنامه، نقش‌ها، مؤسسه | همه موجود | `models/` |

**بدون تغییر باقی می‌مانند (فقط مخفی از کاربر):** موتورهای IRT و FSRS، نسخه‌بندی سؤال،
احراز هویت، نقش‌ها. کاربر هرگز نام این‌ها را نمی‌بیند.

**دستور اجرای سرتاسری (وقتی Docker بالا باشد):**
```bash
make up                                 # بالا آوردن سرویس‌ها
make migrate                            # اعمال مهاجرت‌ها
make seed                               # مؤسسه + ادمین دمو (لازم: importer به یک مؤسسه نیاز دارد)

```

> **نکته‌ی باز (طراحی):** سؤالات عمومی بین همه‌ی آزمون‌ها مشترک‌اند ولی مدل هر سؤال را به یک
> `exam_type` گره می‌زند. مجموعه‌ی شروع فعلاً `executive` برچسب خورده. گزینه‌ها برای حلِ واقعی:
> (الف) تکرارِ عمومی‌ها برای هر آزمون، (ب) مجاز کردن `exam_type=NULL` برای «عمومیِ مشترک» و
> فیلترِ practice طوری که عمومی‌ها در همه‌ی آزمون‌ها بیایند، (ج) جدولِ واسط. → تصمیم در فاز ۴/۵.
